Guía básica: GitHub Pages para alojamiento de sitios web

¿Qué es GitHub Pages?

GitHub Pages es un servicio gratuito de alojamiento web ofrecido por GitHub que permite publicar sitios web directamente desde un repositorio. Está diseñado principalmente para contenido estático, como páginas hechas con HTML, CSS y JavaScript.

¿A quién está dirigido?

GitHub Pages es ideal para:

  • Estudiantes y principiantes en desarrollo web
  • Desarrolladores frontend
  • Personas que quieren crear portafolios personales
  • Proyectos de documentación
  • Blogs técnicos
  • Proyectos open source

No está pensado para aplicaciones complejas que dependan de backend.

¿Qué puedes hacer con GitHub Pages?

  • Publicar sitios web estáticos
  • Crear portafolios personales
  • Desplegar documentación técnica
  • Crear blogs (usando generadores como Jekyll o Hugo)
  • Compartir proyectos frontend

¿Qué NO puedes hacer?

GitHub Pages NO soporta backend, lo que significa:

  • No puedes ejecutar código del lado del servidor (PHP, Python, Node.js, etc.)
  • No puedes usar bases de datos directamente
  • No puedes crear APIs propias dentro del mismo servicio

Si necesitas backend, debes usar servicios externos (como APIs o servidores propios).

Requisitos básicos para funcionar

Para que un sitio funcione en GitHub Pages necesitas:

  • Un repositorio en GitHub

  • Un archivo principal llamado:

    • index.html (más común)
    • o index.md (si usas Jekyll)
  • Archivos estáticos adicionales (CSS, JS, imágenes, etc.)

Tecnologías que puedes usar

GitHub Pages soporta:

  • HTML
  • CSS
  • JavaScript

También puedes usar:

  • Frameworks frontend (React, Vue, etc. — con build previo)

  • Generadores de sitios estáticos como:

    • Jekyll (soporte nativo)
    • Hugo (requiere build previo)

Nota sobre el “mapeo”

GitHub Pages sirve los archivos directamente desde el repositorio:

  • /index.html → página principal
  • /about.html → tusitio/about.html
  • /assets/style.css → recursos estáticos

Página de ejemplo

Para este ejemplo, vamos a crear una página web, sencilla, con la siguiente estructura y código fuente.

Estructura del proyecto

/TU-REPOSITORIO
 ├── index.html
 ├── style.css
 └── script.js

index.html

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Mi página en GitHub Pages</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Hola mundo 🌐</h1>
    <p>Mi primera página en GitHub Pages</p>
    <button onclick="saludo()">Haz clic</button>

    <script src="script.js"></script>
</body>
</html>

style.css

body {
    font-family: Arial, sans-serif;
    text-align: center;
    background-color: #f4f4f4;
}

h1 {
    color: #333;
}

script.js

function saludo() {
    alert("¡Funciona JavaScript!");
}

Creación del repositorio

Para crear el repositorio, seguimos los siguientes pasos.

  1. Entramos a GitHub y creamos nuestro repositorio. Para el nombre del repositorio, usare ejemplo como nombre.

  2. Para la visibilidad del repositorio, seleccionamos la opción Public, para que sea pública y desactivamos la opción de crear un README automáticamente.

  3. Hacemos clic en el botón verde Create repository

Subir archivos al repositorio

Ahora, para subir nuestro contenido al repositorio, tenemos algunas opciones.

Opción 1: Desde terminal (Git)

git init
git add .
git commit -m "Primer commit"
git branch -M main
git remote add origin https://github.com/usuario/repositorio.git
git push -u origin main

Opción 2: Desde la página web

  1. Entramos a nuestro repositorio
  2. Hacemos clic en uploading an existing file
  3. Nos llevará a otra página donde podemos subir nuestros archivos. Una vez subidos, debemos darle un nombre al commit y una descripción.
  4. Damos clic en Commit changes

Activar GitHub Pages

  1. Ir al repositorio
  2. Entrar a Settings
  3. Buscar la sección Pages
  4. En la sección “Branch”:
    • Da clic en noney cambialo a main
    • Asegurate de tener selecionado el directorio /root
  5. Para guardar los cambios, dale clic a save

Después de unos segundos/minutos, tu sitio estará disponible en:

https://usuario.github.io/repositorio/

En mi ejemplo, aparece como https://theredwolf-dev.github.io/ejemplo/

Resultado esperado

Una página web simple que:

  • Muestra un título
  • Tiene estilos básicos
  • Ejecuta JavaScript al hacer clic

Siguientes pasos

GitHub Pages es una excelente herramienta para comenzar en el desarrollo web y publicar proyectos sin costo. Es simple, rápido y suficiente para la mayoría de proyectos frontend, aunque con limitaciones importantes en backend que deben considerarse desde el inicio.

Aunque en este ejemplo usamos HTML, CSS y JavaScript, existen herramientas más avanzadas para la creación de sitios estáticos como Jekyll y Hugo, que facilitan la organización y generación de contenido de forma más eficiente.

Además, es posible mejorar la presentación del sitio mediante dominios personalizados y servicios como Cloudflare. Sin embargo, en este punto es donde termina lo completamente gratuito, ya que adquirir un dominio propio implica un costo.

En próximos artículos veremos cómo usar Jekyll y Hugo, así como la configuración de dominios personalizados para llevar tu proyecto a un nivel más profesional.
