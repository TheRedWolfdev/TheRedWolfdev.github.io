Cloudflare Tunnel: Expón tu localhost a Internet

Cuando desarrollamos una API o una aplicación web, es común que necesitemos hacerla accesible desde Internet para realizar pruebas, compartir avances o integrar servicios externos.

Si el proyecto es únicamente Front-End, el problema es sencillo de resolver: plataformas como GitHub Pages permiten desplegar sitios estáticos de forma rápida y gratuita.

Sin embargo, la situación cambia cuando necesitamos un Back-End. En este caso, ya no basta con un hosting estático, y las opciones gratuitas suelen ser limitadas o implican configuraciones más complejas.

Aquí es donde entra una alternativa interesante: utilizar nuestra propia máquina como servidor.

Cloudflare Tunnel

Cloudflare Tunnel es un servicio gratuito y seguro que permite exponer aplicaciones locales (como servidores web, APIs o dashboards) a Internet sin necesidad de abrir puertos en el router ni configurar port forwarding.

En lugar de exponer directamente tu red, Cloudflare crea un túnel seguro desde tu máquina hacia su infraestructura. A partir de ahí, actúa como un proxy que recibe las solicitudes desde Internet y las redirige a tu aplicación local.

En términos simples:

  • Tu aplicación sigue corriendo en localhost
  • Cloudflare establece una conexión saliente segura
  • Los usuarios acceden a tu servicio mediante una URL pública gestionada por Cloudflare

Esto no solo simplifica la configuración, sino que también mejora la seguridad, ya que tu red no queda expuesta directamente.

Ventajas de Cloudflare Tunnel

La principal ventaja de usar Cloudflare Tunnel es su facilidad de uso. Basta con ejecutar unos cuantos comandos para exponer tu aplicación, evitando la necesidad de abrir puertos o configurar el router doméstico.

Otra ventaja importante es la seguridad. El túnel que se establece hacia la infraestructura de Cloudflare está completamente cifrado, lo que añade una capa de confidencialidad al tráfico de datos. Nota: esto protege el canal de comunicación, pero no asegura tu aplicación. Vulnerabilidades como XSS, LFI o fallos en la configuración del sistema siguen siendo responsabilidad del desarrollador.

Además, permite trabajar con dominios y subdominios personalizados, así como contar con HTTPS automático, proporcionando certificados válidos sin configuraciones adicionales.

Desventajas de Cloudflare Tunnel

Una de las principales desventajas es que tu proyecto depende completamente de tu máquina y tu conexión a Internet. Si necesitas que el servicio esté disponible 24/7, tu equipo también deberá estar encendido todo el tiempo, lo cual no es ideal para proyectos más serios.

Otra limitación importante es que no está pensado para cargas altas. Si tu aplicación requiere mucho procesamiento o recibe tráfico constante, tu propia máquina puede convertirse en el cuello de botella e incluso colapsar.

Además, al depender de un intermediario como Cloudflare, también existe una dependencia externa: cualquier problema en el servicio afectará directamente la disponibilidad de tu aplicación.

Instalación de Cloudflare Tunnel

Ahora bien, vamos a instalar el software de Cloudflare llamado Cloudflared; sí solo cambia por la d al final. La instalación va a depender de tu Sistema Operativo, si lo deseas, aquí te dejo la guía oficial de instalación de cloudflared proporcionada por Cloudflare. Yo, por mi parte, voy a continuar la instalación para sistemas Ubuntu.

Comandos de instalación

A continuación se mostrarán los comandos necesarios para la instalación de Cloudflared.

curl -fsSL https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg >/dev/null
echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/cloudflared.list
sudo apt update
sudo apt install -y cloudflared

Explicación de los comandos

Descargar e instalar la clave GPG del repositorio

curl -fsSL https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg >/dev/null

Este comando descarga la clave GPG oficial de Cloudflare y la guarda en el sistema.

curl -fsSL: descarga el archivo de forma silenciosa y segura | sudo tee ...: guarda el contenido en /usr/share/keyrings/ con permisos de administrador >/dev/null: evita mostrar salida en pantalla

La clave GPG permite verificar que los paquetes instalados provienen realmente de Cloudflare y no han sido alterados.

Agregar el repositorio oficial de cloudflared

echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/cloudflared.list

Este comando añade el repositorio oficial al sistema.

echo "deb ...": define la fuente del repositorio signed-by=...: indica qué clave usar para validar los paquetes $(lsb_release -cs): inserta automáticamente el nombre de tu versión de Ubuntu (por ejemplo: focal, jammy) tee ...cloudflared.list: guarda el repositorio en la lista de fuentes de APT

Permite que apt pueda encontrar e instalar cloudflared directamente desde los servidores oficiales.

Actualizar la lista de paquetes

sudo apt update

Este comando actualiza la lista de paquetes disponibles, incluyendo el nuevo repositorio que acabas de agregar. Sin esto, el sistema no sabría que existe cloudflared.

Instalar cloudflared

sudo apt install -y cloudflared

Este comando instala la herramienta en tu sistema.

-y: acepta automáticamente las confirmaciones

Una vez terminado, ya puedes usar cloudflared desde la terminal.

Verificar instalación

Para verificar si tu instalación funciona correctamente, puedes usar el siguiente comando.

cloudflared --version

En caso que no haya funcionado, verifica que hayas realizado bien los pasos de instalación antes de continuar con el tutorial.

Creación del túnel

Antes de crear tu primer túnel, es importante cumplir con dos requisitos básicos:

  1. Estar ubicado en el directorio de tu proyecto web
  2. Tener un servicio HTTP local en ejecución

Esto se debe a que el túnel va a redirigir el tráfico hacia un servicio que ya debe estar activo en tu máquina.

Levantar un servidor HTTP local

Para este ejemplo, utilizaremos Python, ya que incluye un modulo de servidor HTTP simple integrado.

Puedes iniciarlo con el siguiente comando:

python3 -m http.server 8080

Con este comando levantamos un servicio HTTP en nuestro directorio actual de trabajo por el puerto 8080, puedes elegir otro puerto pero ten en cuenta que deberás usar ese mismo puerto en el siguiente comando. Con nuestro servicio corriendo, vamos a activar el túnel. Para ello ejecutamos el soguiente comando.

cloudflared tunnel --url http:localhost:8080

Nota: En caso que hayas elegido otro puerto, cambia el puerto 8080 por el puerto que elegiste.

Una vez que ejecutes el comando, Clouadflered te entregará en la terminal una URL pública con el que podrás entrar a tu proyecto.

https://random-subdomain.trycloudflare.com

Conclusión

Aun así, Cloudflare Tunnel ofrece muchas más posibilidades. Es posible crear túneles persistentes, utilizar dominios personalizados, configurar subdominios e incluso mantener servicios más estables y profesionales.
